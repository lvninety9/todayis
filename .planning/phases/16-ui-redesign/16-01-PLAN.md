# Wave 1: Design System Foundation (UI-10)

**Goal**: CSS variables 확장, 토큰 시스템 구축, dark mode 완성도 향상

**must_haves**:
- 모든 페이지가 CSS variables를 통해 일관된 색상/타이포그래피/스페이스 사용
- Dark mode가 모든 페이지에서 올바르게 동작
- Contrast ratio가 WCAG AA 기준(4.5:1)을 충족

## Task 1: CSS Variables 확장 및 토큰 시스템

<task>
<read_first>
- src/app/globals.css
</read_first>

<acceptance_criteria>
- globals.css에 --color-surface token 정의됨
- globals.css에 --shadow-sm, --shadow-md, --shadow-lg, --shadow-xl 정의됨
- globals.css에 --radius-sm, --radius-md, --radius-lg 정의됨
- dark mode variant에 모든 새 token 대응 값 포함
</acceptance_criteria>

<action>
globals.css에 다음 tokens 추가:

```css
/* Surface colors */
--color-surface: hsl(30, 20%, 98%);
--color-surface-raised: hsl(30, 15%, 100%);

/* Border */
--color-border: hsl(30, 15%, 85%);

/* Shadows */
--shadow-sm: 0 1px 2px 0 hsl(0 0% 0% / 0.05);
--shadow-md: 0 4px 6px -1px hsl(0 0% 0% / 0.1);
--shadow-lg: 0 10px 15px -3px hsl(0 0% 0% / 0.1);
```

dark mode에서:
```css
--color-surface: hsl(30, 15%, 10%);
--color-surface-raised: hsl(30, 15%, 15%);
--color-border: hsl(30, 15%, 25%);
```
</action>
</task>

## Task 4: Contrast Ratio 검증

<task>
<read_first>
- src/components/ui/button.tsx
- src/components/ui/input.tsx
</read_first>

<acceptance_criteria>
- 모든 버튼 텍스트의 contrast ratio >= 4.5:1
- 모든 입력 필드 라벨의 contrast ratio >= 4.5:1
- disabled 상태의 contrast ratio >= 3:1
</acceptance_criteria>

<action>
button.tsx에서:
- primary variant의 text 색상이 --color-surface 위에서 4.5:1 충족하는가 확인
- 필요시 색상 조정 (예: --primary를 더 진하게)

input.tsx에서:
- label 색상이 --color-border 이상으로 충분히 진한가确认
- placeholder 색상이 3:1 충족하는가确认
</action>
</task>
